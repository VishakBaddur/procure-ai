## Vendor 1 (SafetyPro) – **Line items correct; total was wrong (now fixed)**

**Source (PDF content):**
- Row 1: SafetyPro G-200 Impact-Resistant Safety Goggles, 500, $21.50, $10,750  
- Row 2: SafetyPro H-150 ANSI Z89.1 Hard Hats (White), 500, $26.00, $13,000  
- Row 3: SafetyPro V-90 Hi-Vis Mesh Vest (Class 2), 500, $18.00, $9,000  
- **Subtotal:** $32,750 → **Bulk discount (8%):** -$2,620 → **Total (USD):** $30,130  
- Payment: Net 30; Warranty: 2 years on all items  

**UI (your screenshot – Document "1") before fix:**
- **Total shown:** USD 13,000.00 ❌ — This is only Product 2’s line total, not the document total.  
- **Product 1:** Safety Goggles, USD 21.50, 500–500, Total USD 10,750.00 ✅  
- **Product 2:** Hard Hats, USD 26.00, 500–500, Total USD 13,000.00 ✅  
- **Product 3:** Hi-Vis Mesh Vest, USD 18.00, 500–500, Total USD 9,000.00 ✅  

**Cause:** The backend was using the LLM’s `summary.total_price_range.max` (which sometimes picked a single line total, e.g. 13000) instead of the **sum of line-item totals** (10,750 + 13,000 + 9,000 = **32,750**).

**Fix applied:** In `price_comparison_agent.py`, the displayed total now **prefers the summed total** from all products. We only fall back to `summary.total_price_range.max` when the summed total is zero. So Vendor 1 will now show **USD 32,750.00** (sum of the three line items).  
*(Note: The PDF’s final total after discount is $30,130; the UI currently shows the sum of line totals. If you want to show post-discount total, the LLM would need to extract Subtotal vs discount vs Total (USD) and we’d expose that.)*
